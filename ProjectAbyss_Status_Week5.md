## 1. 5주차 목표

5주차의 핵심 목표는 1스테이지 프로토타입을 넘어, 스테이지 진행 구조와 2스테이지 기반을 연결하는 것이다.

현재 Day 24~27까지 다음 작업이 완료된 상태다.

- 보스 피격/페이즈2 색상 피드백 정리
- 카피 스킬 슬롯 HUD 추가
- 스테이지 전환 골격 추가
- 압력 시스템을 StageData 기반으로 정리
- 압력 저항은 페널티만 줄이고, 경험치 보상은 유지되도록 변경
- 돌연변이 2종 추가
- 의태 기관을 기획 문서 방향으로 변경
- 기본 스킬 Slash가 시작 시 Lv1로 등록되도록 수정

5주차는 다음 흐름을 완성하는 데 집중한다.

```text
1스테이지 보스 처치
→ 카피 스킬 선택
→ 검은 화면 페이드 아웃
→ 다음 스테이지 이름 표시
→ 씬 또는 스테이지 상태 전환
→ 페이드 인
→ 2스테이지 시작
```

## 2. 스테이지 전환 방향

보스 처치 후 카피 스킬을 선택하면 즉시 끊기지 않고, 검은 화면으로 페이드 아웃된다.

페이드 중 다음 스테이지 이름을 표시하고, 이후 페이드 인으로 새 스테이지를 시작한다.

목표는 메뉴식 결과 화면보다 몰입감 있는 연속 진행이다.

구현 후보 흐름:

1. `CopySkillSelectCardUI.OnClick()`
2. `CopySkillManager.AssignSkill()`
3. `CopySkillSelectPanel.Hide()`
4. `GameManager.TriggerStageClear()`
5. `StageManager.TransitionToNext()`
6. `StageTransitionUI.PlayFadeOut(nextStage.displayName, callback)`
7. 씬 로드 또는 같은 씬 내 스테이지 데이터 교체
8. `PlayerProgressData.Restore()`
9. `StageTransitionUI.PlayFadeIn()`

## 10. 주의할 점

- `StageTransitionUI`는 현재 스켈레톤 상태이므로, 페이드 구현 전까지 빌드 경고가 남을 수 있다.
- `PlayerProgressData`는 아직 실제 저장/복원이 구현되지 않았다.
- 카피 스킬은 현재 1스테이지 3종 중심이므로, 2스테이지 스킬 추가 시 `CopySkillID`와 `FindSkillComponent()` 확장이 필요하다.
- `GameManager`, `StageManager`, `PlayerProgressData`, `StageTransitionUI`의 책임이 겹치지 않도록 유지한다.
- 압력 저항은 페널티 감소 전용이며, 경험치 보상은 `RawPressure` 기준을 유지한다.
- 스테이지 전환 중 `Time.timeScale`이 0일 수 있으므로 UI 페이드는 반드시 실시간 기준으로 처리한다.
