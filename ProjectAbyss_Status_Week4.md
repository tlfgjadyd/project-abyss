# Project: Abyss - 진행 상황 인수인계 문서

작성일: 2026-05-06

## 1. 프로젝트 개요

Project: Abyss는 Unity 2022 LTS 기반의 2D 탑다운 뱀서라이크 생존 액션 게임이다.

핵심 컨셉은 심해 연구소에서 탈출한 생체 실험체가 적을 처치하고 능력을 흡수하며 성장하고, 최종적으로 연구소장에게 복수하는 구조다.

현재 개발 방향은 그래픽보다 핵심 플레이 루프와 시스템 구현을 우선하는 화이트박스 프로토타입이다.

## 2. 핵심 게임 구조

### 기본 플레이 루프

1. 플레이어가 WASD로 이동한다.
2. 자동 공격이 주변 적을 탐색한다.
3. 가장 가까운 적을 대상으로 공격한다.
4. 적이 피해를 받고 사망한다.
5. 적 사망 시 경험치 오브와 생체 에너지를 지급한다.
6. 경험치가 차면 레벨업 UI가 뜬다.
7. 스킬을 선택하면 실제 스탯 또는 스킬 성능이 강화된다.
8. 일정 시간이 지나면 보스가 등장한다.
9. 보스를 처치하면 카피 스킬 선택 UI가 뜬다.
10. 카피 스킬을 선택하면 슬롯에 장착되고 스테이지 클리어 상태가 된다.

### 주요 매니저

- `GameManager`
  - 게임 상태 관리
  - `Playing`, `Paused`, `LevelUp`, `Mutation`, `GameOver`, `StageClear`
  - 스테이지 타이머 관리
  - 상태 변경 이벤트 제공

- `LevelManager`
  - 경험치 누적
  - 레벨업 처리
  - 스킬 선택지 제공
  - 압력 시스템의 경험치 배율 반영

- `EnemySpawner`
  - 웨이브별 적 스폰
  - 풀링 기반 적 재사용
  - 최대 활성 적 수 제한
  - 보스 등장 시 비활성화됨

- `ExpOrbPool`
  - 경험치 오브 풀링
  - 적 처치 시 오브 재사용

- `BioEnergyManager`
  - 생체 에너지 관리
  - 최대 에너지 200E
  - 적 처치 시 에너지 획득
  - 카피 스킬 사용 시 에너지 소비

- `CopySkillManager`
  - Q / E / Space 카피 스킬 슬롯 관리
  - 에너지 확인 후 카피 스킬 실행
  - 현재 1스테이지 보스 카피 스킬 3종 구현

- `PressureSystem`
  - 플레이어 Y 좌표 기준 압력 계산
  - 이동속도/공격속도 페널티 적용
  - 경험치 보너스 적용
  - `PlayerStats.EffectiveMoveSpeed`, `PlayerStats.EffectiveAttackSpeed`에 반영

- `MutationManager`
  - 특정 레벨 도달 시 돌연변이 선택 이벤트 준비
  - 현재 기초 구조와 3종 돌연변이 데이터/효과 구현

- `BossSpawner`
  - 일정 시간 후 보스 스폰
  - 보스 등장 시 일반 적 스폰 중지
  - 보스 사망 시 카피 스킬 선택 패널 표시

## 3. 플레이어 구조

### `PlayerController`

- WASD 이동
- `Rigidbody2D` 기반 이동
- `PlayerStats.EffectiveMoveSpeed` 사용
- 마지막 이동 방향 저장
- 대시 중 일반 이동 정지

### `PlayerStats`

- 최대 HP
- 현재 HP
- 이동속도
- 공격력
- 공격속도
- HP 재생
- 압력 보정 배율
- 무적 여부
- 회복 차단 여부

현재 압력 시스템 때문에 직접 `moveSpeed`, `attackSpeed`를 쓰지 않고 다음 값을 사용하는 것이 원칙이다.

- `EffectiveMoveSpeed`
- `EffectiveAttackSpeed`

### `PlayerCombat`

- `Physics2D.OverlapCircleAll`로 적 탐색
- 가장 가까운 적 자동 타겟팅
- `Slash` 실행
- 감지 범위는 돌연변이로 배율 증가 가능

## 4. 일반 스킬 구현 현황

### `Slash`

- 기본 근접 부채꼴 공격
- 범위 내 적 중 공격 방향 각도 안의 대상에게 피해
- 레벨업 시 범위/공격력/넉백 강화 가능

### `PoisonNeedle`

- 가까운 적에게 투사체 발사
- 투사체 풀링 사용
- 다중 발사와 관통 구조 구현
- 투사체는 태그가 아니라 `IDamageable` 기준으로 피해 적용
- 관통 시 같은 적 중복 타격 방지 구현

### `BioticExplosion`

- 플레이어 중심 원형 범위 공격
- 적에게 피해
- 스턴 적용 가능

### `ElectricEngine`

- 가까운 적 1명에게 전기 피해
- 주변 적에게 연쇄 피해

## 5. 적 / 보스 구조

### 일반 적

구현된 일반 적 데이터/프리팹:

- `Enemy_Default`
- `Enemy_Guard`
- `Enemy_Shooter`

현재 `Enemy_Shooter`는 이름만 Shooter이며, 실제 원거리 공격 AI는 아직 구현되지 않았다.

### `EnemyBase`

- HP 관리
- 피격 처리
- 사망 처리
- 경험치 오브 드롭
- 생체 에너지 지급

### `EnemyAI`

- 플레이어 추적 이동
- 스턴 상태에서는 정지

### 보스

현재 1스테이지 보스 프로토타입 구현 완료.

보스 프리팹:

- `Boss_ExperimentalSubjects`

보스 데이터:

- `Boss_ExperimentalSubjects.asset`

### `BossBase`

- 보스 HP 관리
- 피격 처리
- 페이즈2 진입 이벤트
- 사망 이벤트
- 접촉 피해

### `BossAI`

- 플레이어 추적
- 페이즈2에서 이동속도 증가
- 일정 간격으로 돌진

### `BossSpawner`

- 현재 480초에 보스 등장
- 보스 등장 시 `EnemySpawner` 비활성화
- 보스 사망 시 `CopySkillSelectPanel` 표시

### `BossHPBar`

- 보스 등장 중에만 표시
- 보스 이름, HP 슬라이더, HP 텍스트 표시

## 6. 카피 스킬 시스템

보스 처치 후 보스 데이터의 `copySkillOptions`에 등록된 카피 스킬 3개 중 하나를 선택한다.

현재 1스테이지 보스 카피 스킬:

- `Berserk`
  - 이동속도 증가
  - 공격속도 증가
  - 에너지 50E

- `Dash`
  - 바라보는 방향으로 짧은 돌진
  - 돌진 중 무적
  - 에너지 60E
  - 대시 중 재입력 시 에너지가 소모되지 않도록 `CanExecute()` 구현

- `HealingFactor`
  - 최대 체력 비례 회복
  - 에너지 30E

카피 스킬 데이터에는 `copySkillID`가 필요하다.

- Berserk -> `Berserk`
- Dash -> `Dash`
- HealingFactor -> `HealingFactor`

## 7. 돌연변이 시스템 기초

현재 돌연변이 시스템은 기초 구현 상태다.

구현된 돌연변이:

- `Overload`
  - 공격력 증가
  - 최대 HP 감소

- `OvgrownTentacle`
  - 스킬 범위 증가
  - 이동속도 감소

- `MimicryOrgan`
  - 카피 스킬 에너지 소모 제거
  - HP 회복 차단

주의: `OvgrownTentacle`은 오타가 있는 이름으로 구현되어 있다. 추후 여유가 있으면 `OvergrownTentacle`로 정리하는 것을 권장한다.

  - EXP
  - 생체 에너지
  - 레벨
  - 타이머
  - 압력 표시

- LevelUpPanel
  - 일반 스킬 선택

- GameOverPanel
  - 게임오버 / 스테이지 클리어 표시
  - 재시작 버튼

- BossHPBar
  - 보스 전용 HP바

- CopySkillSelectPanel
  - 보스 처치 후 카피 스킬 선택

- MutationPanel
  - 돌연변이 선택

## 9. 현재 주의할 점

- `CopySkillManger` 오브젝트 이름에 오타가 남아 있다.
  - 기능 문제는 아니지만 `CopySkillManager`로 수정 권장

- `Boss_ExperimentalSubjects.prefab`의 `Rigidbody2D` gravity scale이 프리팹상 1로 보일 수 있다.
  - `BossBase.Awake()`에서 0으로 보정하므로 런타임 문제는 적지만, 프리팹 값도 0으로 맞추면 좋다.

- `PressureSystem`은 현재 테스트를 위해 활성화되어 있을 수 있다.
  - 실제 스테이지 구조에서는 2~3스테이지에서만 활성화 예정

- `MutationData`의 `OvgrownTentacle` 오타가 있다.

- TextMesh Pro / 폰트 에셋 변경량이 크다.
  - 커밋 전에 의도한 변경인지 확인하는 것이 좋다.
